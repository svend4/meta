# Характеристики всех 24 проектов экосистемы Q6

> Граф Q6 — 6-мерный гиперкуб на 64 вершинах (гексаграммах).
> Каждая вершина — 6-битное число 0..63. Каждое ребро — изменение ровно 1 бита.
> Все 24 проекта используют эту структуру как основу.

## 1. hexforth

**Что это:** Стековый язык программирования, где состояние программы — гексаграмма.
6 слотов стека Forth = 6 бит = вершина Q6. Каждое слово языка — переход по ребру графа.
Реализован полный цикл: интерпретатор, компилятор (→ Python / JSON-байткод / DOT-граф), верификатор.

**Для чего применять:**
- Обучение концепциям стековых машин и языков Forth
- Исследование соответствия между вычислительными моделями и комбинаторными структурами
- Генерация тестовых программ для проверки Q6-навигации
- Компиляция в Python-функции для встраивания Q6-логики в другой код
- Академические демонстрации: «вычисление как путь по графу»

**Уникальность:**
Forth существует с 1970-х. Интерпретаторы Forth — тысячи. Но нет ни одной реализации,
где состояние стека изоморфно вершине 6-мерного гиперкуба, а слова языка
строго соответствуют рёбрам этого графа. Это не новый Forth — это новая математическая модель,
для которой Forth служит интерфейсом.

**Существуют ли аналоги:**
- gForth, SwiftForth, RetroForth — традиционные Forth-системы, не связаны с Q6
- Пьезо, Factor, Joy — конкатенативные языки без графовой топологии
- Аналогов, где пространство состояний = гиперкуб, не существует

**Кому поможет:**
- Исследователям языков программирования и вычислительных моделей
- Преподавателям архитектуры ЭВМ и стековых машин
- Разработчикам формальных методов (верификатор программ входит в состав)
- Студентам дискретной математики

**Где есть потребность:**
Ниша «необычных вычислительных моделей» активно развивается (esolang-сообщество,
esoteric programming languages). Академический интерес к конкатенативным языкам
стабильно растёт. Конкретный вопрос «как граф Q6 связан с вычислением» — открытый.

## 2. hexpath

**Что это:** Двухигровая абстрактная стратегия на графе Q6. Доска — 64 узла,
ходы — переходы по рёбрам (изменение 1 черты). Режим игры человек–компьютер,
режим одиночных головоломок. CLI-интерфейс с полным игровым циклом.

**Для чего применять:**
- Игра: самостоятельный продукт для любителей абстрактных стратегий
- Образование: наглядная демонстрация теории графов и метрики Хэмминга
- Исследование: тестирование AI-агентов и алгоритмов поиска пути
- Головоломки: единственный режим — решение задач на Q6 (нахождение пути с ограничениями)
- Прототип для разработчиков настольных игр

**Уникальность:**
Абстрактных стратегий (Go, Chess, Hex, Y) существует очень много.
Но ни одна из них не строится на топологии 6-мерного гиперкуба.
Свойство Q6 — диаметр 6, регулярность степени 6, кодовая структура —
порождает уникальную игровую механику, которой нет нигде больше.
Игра Hex (Piet Hein, 1942) созвучна по названию, но математически не связана.

**Существуют ли аналоги:**
- Игра Hex — на гексагональной сетке, другая математика
- Go, Othello — другая топология
- Numberlink, HexPawn — отдалённо похожие механики, другая структура
- Прямых аналогов на гиперкубе нет

**Кому поможет:**
- Любителям абстрактных настольных игр
- Разработчикам игр, ищущим математически богатую основу
- Исследователям комбинаторной теории игр
- Педагогам, обучающим дискретной математике через игру

**Где есть потребность:**
Рынок абстрактных стратегий стабилен (boardgamegeek.com). Математически обоснованные
игры (Nim, Sprouts) имеют устойчивую академическую аудиторию.
Цифровые реализации для обучения математике — активно востребованное направление.

## 3. hexbio

**Что это:** Библиотека биоинформатики, построенная на изоморфизме генетического кода и Q6.
Кодоны RNA (64 штуки, каждый = 3 нуклеотида × 2 бита) — это ровно 64 вершины Q6.
Точечная мутация = изменение 1 нуклеотида = переход по ребру Q6.
Реализованы: граф мутаций, синонимические пути, RSCU, CAI, wobble-пары, расстояние Грантема.

**Для чего применять:**
- Анализ путей молекулярной эволюции через граф мутаций
- Оценка синонимических и несинонимических замен
- Вычисление CAI (Codon Adaptation Index) для оптимизации экспрессии генов
- Исследование wobble-гипотезы Крика через метрику Q6
- Визуализация генетического кода как математического объекта

**Уникальность:**
Изоморфизм генетического кода и гиперкуба описан в академической литературе
(Jiménez-Montaño, 2009; Rumer, 1966), но готовых библиотек,
реализующих этот изоморфизм как вычислительный инструмент, не существует.
Все существующие инструменты (Biopython, EMBOSS) работают с кодонами как строками,
не используя геометрию Q6.

**Существуют ли аналоги:**
- Biopython — обширная биоинформатика, кодоны как строки, нет Q6
- EMBOSS — набор инструментов, нет графовой топологии
- CodonW, CUSP — только анализ использования кодонов, нет мутационного графа
- Прямого аналога, использующего Q6-изоморфизм, не существует

**Кому поможет:**
- Биоинформатикам, изучающим молекулярную эволюцию
- Синтетическим биологам (оптимизация кодонов)
- Исследователям происхождения генетического кода
- Математикам, изучающим дискретную геометрию в биологии

**Где есть потребность:**
Оптимизация кодонов — активное направление в синтетической биологии (рынок >$10 млрд).
Понимание молекулярной эволюции — фундаментальная задача.
Математические модели генетического кода — открытая исследовательская тема.

## 4. hexspec

**Что это:** Инструмент формальной верификации конечных автоматов через граф Q6.
Каждое состояние автомата = гексаграмма (6 булевых свойств системы).
Верификатор проверяет достижимость, тупики, инварианты, покрытие переходов.
Генератор автоматически строит тестовые сценарии с гарантированным покрытием.

**Для чего применять:**
- Верификация протоколов (TCP, HTTP, пользовательские)
- Проверка корректности конечных автоматов в embedded-системах
- Автоматическая генерация тестовых последовательностей из спецификации
- Поиск deadlock'ов и недостижимых состояний до написания кода
- Документирование протоколов в формате, допускающем машинную проверку

**Уникальность:**
Model checking — зрелая область (SPIN, TLA+, Alloy). Но все инструменты работают
с произвольными автоматами. hexspec накладывает ограничение: переходы = изменение
ровно 1 бита. Это ограничение отсекает целый класс ошибок (нарушение атомарности)
и позволяет использовать геометрию Q6 для поиска путей.

**Существуют ли аналоги:**
- SPIN/Promela — полноценный model checker, без Q6-ограничений
- TLA+ — мощный, сложный в освоении
- Alloy Analyzer — реляционная логика, не конечные автоматы
- FSM-библиотеки (transitions, pytransitions) — нет верификации, нет Q6
- Прямого аналога нет

**Кому поможет:**
- QA-инженерам и тест-архитекторам
- Разработчикам сетевых протоколов
- Embedded-разработчикам (прошивки с конечными автоматами)
- Студентам формальных методов

**Где есть потребность:**
Автоматическая генерация тестов из спецификации — горячая тема в QA-сообществе.
Model-based testing (MBT) — активно развивающееся направление.
Инструменты, не требующие изучения Promela/TLA+, востребованы.

## 5. karnaugh6

**Что это:** Минимизатор булевых функций от 6 переменных (64 минтерма).
Реализован алгоритм Куайна–МакКласки. Граф Q6 используется как основа:
его гиперкубическая структура точно совпадает со структурой карт Карно.
Выводит минимальную ДНФ, таблицу импликант, покрытие.

**Для чего применять:**
- Синтез цифровых схем и минимизация логических функций
- Образование: визуализация метода Куайна–МакКласки для 6 переменных
- Проектирование ПЛИС и FPGA (оптимизация логических блоков)
- Верификация результатов других CAD-инструментов
- Академические задачи по дискретной математике

**Уникальность:**
Карты Карно для 4 переменных — стандарт в учебниках. Для 5–6 переменных
визуальный метод неудобен, и студенты и инженеры вынуждены использовать
специализированные CAD-системы. Данный инструмент закрывает именно этот пробел:
лёгкий CLI без зависимостей, чистый Куайн–МакКласки, понятный вывод.

**Существуют ли аналоги:**
- Логисим, Quine McCluskey online — есть веб-инструменты, но с ограничениями UI
- ABC (Berkeley) — мощный, но тяжёлый CAD для синтеза схем
- Espresso — быстрый эвристический минимизатор, не точный
- Лёгких CLI-инструментов именно для 6 переменных с объяснением — мало

**Кому поможет:**
- Студентам цифровой схемотехники и дискретной математики
- Инженерам, проектирующим логические схемы без CAD
- Преподавателям, демонстрирующим алгоритм Куайна–МакКласки
- Разработчикам FPGA, нуждающимся в быстрой проверке

**Где есть потребность:**
Каждый год миллионы студентов изучают цифровую схемотехнику.
Простые, работающие без установки инструменты для минимизации — стабильный спрос.

## 6. hexca

**Что это:** Клеточный автомат на графе Q6, где каждая клетка имеет 64 состояния
(вместо 2 в классическом CA Вольфрама). Поддерживает 1D-ленту и 2D-тор
с периодическими граничными условиями. Несколько встроенных правил (XOR, majority vote,
Conway-подобные). ASCII-анимация в терминале.

**Для чего применять:**
- Исследование сложных динамических систем с богатым пространством правил
- Поиск новых правил с интересным поведением (аттракторы, паттерны)
- Образовательные демонстрации теории клеточных автоматов
- Генерация псевдослучайных последовательностей
- Прототипирование моделей сложных систем

**Уникальность:**
Автоматы Вольфрама (2 состояния, 256 правил) — хорошо изучены.
Game of Life (2 состояния) — классика. Автоматы с большим числом состояний
(64 = 2⁶) исследуются значительно меньше, а их пространство правил
астрономически огромно. Использование Q6 как структуры переходов
(только соседние гексаграммы, изменение 1 черты) — уникальное ограничение.

**Существуют ли аналоги:**
- Golly — мощный симулятор CA, поддерживает многосостояния
- Mirek's Cellebration — широкий набор правил CA
- Langton's Ant, Turmites — другие многосостоятельные CA
- Ни один из них не использует Q6 как пространство состояний

**Кому поможет:**
- Исследователям сложных систем и эмерджентного поведения
- Студентам, изучающим CA и дискретные динамические системы
- Разработчикам процедурной генерации контента в играх
- Криптографам, ищущим новые источники псевдослучайности

**Где есть потребность:**
Клеточные автоматы переживают ренессанс интереса (Wolfram Physics Project, 2020).
Процедурная генерация — активно используется в игровой индустрии.
Образовательные симуляторы — стабильный спрос.

## 7. hexcrypt

**Что это:** Криптографические примитивы, построенные на Q6. S-блоки (аффинные,
случайные, оптимизированные), анализ дифференциальных и линейных характеристик
(DDT и LAT таблицы), потоковый шифр на основе Q6, сеть Фейстеля.
Поиск S-блоков с максимальной нелинейностью.

**Для чего применять:**
- Исследование криптографических свойств булевых перестановок
- Проектирование S-блоков для симметричных шифров
- Образование: демонстрация дифференциального и линейного криптоанализа
- Прототипирование лёгких шифров для constrained devices
- Академические исследования: связь bent-функций, нелинейности и Q6

**Уникальность:**
Криптографических библиотек много. Но анализ S-блоков в контексте геометрии Q6 —
нестандартный угол зрения. В частности: bent-функции на Q6 связаны с оптимальными
S-блоками, и эта связь здесь реализована как вычислительный инструмент.
Поиск S-блоков с заданными дифференциальными характеристиками через Q6 — редкость.

**Существуют ли аналоги:**
- SageMath — имеет cryptographic tools, но без Q6-фокуса
- CryptoMiniSat — SAT-based, другой подход
- специализированные библиотеки (pycryptodome) — реализация, не анализ
- Инструментов для проектирования S-блоков через гиперкуб нет

**Кому поможет:**
- Исследователям симметричной криптографии
- Разработчикам лёгких шифров (IoT, embedded)
- Студентам курсов прикладной криптографии
- Специалистам по анализу шифров

**Где есть потребность:**
Проектирование лёгких шифров (lightweight cryptography) — активное направление
(стандарт NIST LWC завершён в 2023). Анализ S-блоков — регулярная задача
при разработке новых шифров.

## 8. hexlearn

**Что это:** Алгоритмы машинного обучения, адаптированные под метрическое пространство Q6.
k-NN с метрикой Хэмминга, k-медоиды (k-means для дискретных пространств),
наивный байесовский классификатор, марковские цепи на Q6, спектральное вложение.

**Для чего применять:**
- Кластеризация дискретных 6-битных данных
- Классификация объектов, естественно представимых 6 булевыми признаками
- Обучение ML на «нестандартных» метрических пространствах
- Анализ марковских процессов на гиперкубе
- Исследование: как стандартные ML-алгоритмы ведут себя на Q6

**Уникальность:**
scikit-learn поддерживает метрику Хэмминга. Но адаптация всего пайплайна
(кластеризация, марковские цепи, байесовский вывод) под конкретное
дискретное пространство Q6 с использованием его топологии — нестандартна.
k-медоиды для Q6 с гарантиями на медоиды как вершины Q6 — специфично.

**Существуют ли аналоги:**
- scikit-learn — общий ML, поддерживает Хэмминг, но не Q6-специфичен
- networkx + ML — возможно, но нет готового решения
- Специализированных ML-библиотек для гиперкубов нет

**Кому поможет:**
- Исследователям, работающим с дискретными пространствами признаков
- Специалистам по анализу геномных данных (6-битные кодоны — прямое применение)
- Студентам, изучающим ML на нестандартных пространствах
- Разработчикам рекомендательных систем с категориальными признаками

**Где есть потребность:**
ML на дискретных и графовых пространствах — активная область (Graph Neural Networks).
Кластеризация категориальных данных — фундаментальная задача в data science.

## 9. hexopt

след, норма, след-билинейная форма, порядки элементов, примитивные элементы,
циклотомические классы, минимальные многочлены, BCH-коды, логарифм Зеха.

**Для чего применять:**
- Кодирование: построение BCH и Reed-Solomon кодов над GF(64)
- Криптография: конструкция S-блоков через поле Галуа
- Образование: конкретный пример конечного поля с 64 элементами
- Исследование: связь между GF(2⁶) и геометрией Q6
- Алгебраическая теория кодирования

**Уникальность:**
GF(2^n) реализован во многих системах. Но конкретно GF(2⁶) с полным набором
аналитических инструментов (все 14 примитивных элементов, все циклотомические классы,
BCH-генераторы для произвольного расстояния) и явной связью с Q6 — нестандартно.

**Существуют ли аналоги:**
- SageMath GF(2^6) — есть, полноценный
- Специализированных standalone инструментов без SageMath — мало

**Кому поможет:**
- Студентам абстрактной алгебры и теории кодирования
- Криптографам, проектирующим AES-подобные схемы
- Исследователям алгебраической теории кодирования

**Где есть потребность:**
GF(2^n) — основа AES (n=8), Reed-Solomon кодов (QR-коды, RAID, DVD),
LDPC кодов. Образовательные инструменты стабильно востребованы.

## 21. hexmat

**Что это:** Линейная алгебра над GF(2) на 6-мерном пространстве.
Матрицы 6×6 над GF(2), ранг, ядро, образ, обратная матрица,
группа GL(6,2) (порядок 20158709760), генераторные и проверочные матрицы линейных кодов,
разложение Гаусса над GF(2).

**Для чего применять:**
- Построение и анализ линейных кодов (генераторные матрицы)
- Исследование группы GL(6,2) — группы автоморфизмов пространства
- Решение систем линейных уравнений над GF(2)
- Образование: линейная алгебра над двоичным полем
- Связь с hexcode и hexring через матричное представление

**Уникальность:**
NumPy не поддерживает GF(2) арифметику напрямую.
SageMath поддерживает, но требует тяжёлой установки.
hexmat — лёгкая реализация без зависимостей, специфичная для 6D.

**Существуют ли аналоги:**
- SageMath MatrixSpace(GF(2), 6) — есть, но тяжёлый
- galois (Python library) — есть GF(2) матрицы, более общий
- Лёгких standalone реализаций для GF(2)⁶ мало

**Кому поможет:**
- Студентам алгебры и теории кодирования
- Разработчикам кодов коррекции ошибок
- Исследователям, нуждающимся в лёгкой GF(2) библиотеке

## 22. hexlat

**Что это:** Q6 как булева решётка B₆ (частично упорядоченное множество).
Операции meet/join, частичный порядок (⊆ по битам), функция Мёбиуса,
ζ-функция, обращение Мёбиуса, цепи, антицепи (теорема Дилворта),
число Уитни, многочлен Пуанкаре, характеристический многочлен решётки,
разложение Мирского.

**Для чего применять:**
- Комбинаторный анализ структуры подмножеств 6-элементного множества
- Топологический анализ решётки (эйлерова характеристика, гомологии)
- Исследование функции Мёбиуса (включения-исключения на Q6)
- Образование: теория решёток на конкретном примере
- Приложения в теории кодирования: максимальные антицепи = коды с max Хэммингом

**Уникальность:**
Теория решёток (SageMath Posets) реализована. Но конкретно B₆ со всеми
специфическими инструментами (разложение Мирского, ζ-полином, числа Уитни,
связь с Q6-геометрией) — специфичная реализация.

**Существуют ли аналоги:**
- SageMath Posets — полноценный, но общий
- Специализированных инструментов для B₆ нет

**Кому поможет:**
- Комбинаторикам и алгебраистам
- Студентам дискретной математики
- Исследователям топологических инвариантов решёток

## 23. hexsym

**Что это:** Полный анализ группы автоморфизмов Aut(Q6) = B₆ = S₆ ⋉ (Z₂)⁶
порядка 46080. Орбиты под действием группы, лемма Бёрнсайда,
индекс Пойа, количество различных раскрасок Q6 с n цветами,
орбиты рёбер, цикловой индекс, стабилизаторы.

**Для чего применять:**
- Подсчёт структурно различных конфигураций на Q6 с учётом симметрии
- Каноническая форма гексаграммы (представитель орбиты)
- Образование: лемма Бёрнсайда и теорема Пойа на конкретном примере
- Применение в химии: изомеры молекул с 6 бинарными позициями
- Криптография: симметрия S-блоков

**Уникальность:**
Группа Aut(Q_n) известна теоретически. Но вычислительный инструмент,
который перечисляет орбиты, вычисляет цикловые индексы и применяет
теорему Пойа конкретно для Q6 — нестандартный.

**Существуют ли аналоги:**
- SageMath PermutationGroup — есть, но не специфичен для Q6
- GAP (Groups, Algorithms, Programming) — мощный, тяжёлый

**Кому поможет:**
- Математикам, изучающим теорию групп и комбинаторику
- Химикам (подсчёт изомеров)
- Криптографам (анализ симметрии S-блоков)
- Студентам абстрактной алгебры

## 24. hexnav

**Что это:** Интерактивный CLI навигатор по графу Q6. Базовый инструмент экосистемы:
показывает текущую гексаграмму в ASCII, 6 соседей, записывает путь,
находит кратчайший путь между двумя узлами, экспортирует путь в JSON.

**Для чего применять:**
- Интерактивное исследование структуры Q6 «вручную»
- Обучение: первое знакомство с графом Q6
- Запись и воспроизведение путей (протоколы навигации)
- Базовый интерфейс для других проектов экосистемы
- Демонстрационный инструмент для презентаций

**Уникальность:**
Это «оболочка» над hexcore. Уникальность — не алгоритмическая, а педагогическая:
единственный инструмент, позволяющий «пройтись» по Q6 шаг за шагом в терминале.

**Существуют ли аналоги:**
Нет, поскольку Q6 — специфичная структура этого проекта.

**Кому поможет:**
- Всем новичкам в экосистеме Q6 (входная точка)
- Преподавателям, демонстрирующим Q6 студентам
- Разработчикам других проектов экосистемы
