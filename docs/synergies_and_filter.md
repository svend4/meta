# Фильтр новых модулей и карта синергий 39 модулей Q6

> Принцип отбора: модуль оправдан если он (1) не существует нигде в стандартных библиотеках,
> (2) даёт вычислительный результат, недостижимый без Q6-контекста,
> (3) органично входит в «часовой механизм» существующих 39 модулей.

### Два новых модуля, которые стоит строить

#### `hexpack` — Упаковки замкнутых полей (Герман, раздел 1+3)

**Почему уникален.** Герман доказал: единственные поля, допускающие плотную упаковку — P = 2^k. Это прямое Q6-утверждение: 64 = 2^6 = наименьший период, содержащий Q6. Ни один существующий модуль не реализует алгоритм упаковки. Результаты не получаются из hexring (WHT), hexgf (поле Галуа) или hexperms (перестановки) — это другая структура.

**Уникальный результат.** Функция `antipode(h)` → пара (h, 63−h) где yang(h)+yang(63−h)=6 — это следствие 2 Германа в Q6-форме. Доказуемо: в Q6 нет фиксированной точки карты h↔63−h (поскольку 63 нечётно, нет решения h=63−h). Это теорема, которой нет ни в одном существующем модуле.

**Команды:**
```
hexpack pack <n>          # кольцо P=n(n-1)/2+1, расставить числа
hexpack fixpoint <P>      # единственная фиксированная точка в кольце
hexpack antipode          # все 32 антиподальных пары Q6: h↔63-h
hexpack packable <P>      # тест: P = 2^k? (плотная упаковка возможна?)
hexpack magic <k>         # магический квадрат из поля P=2^(2k)
hexpack primes            # простые числа в ролях периодов P(n) для n=1..64
```

---

#### `hextrimat` — Треугольная матрица И-Цзин (Андреев)

**Почему уникален.** Существующий hexnav работает со стандартной 8×8 квадратной таблицей гексаграмм. Треугольная (пирамидальная) форма — принципиально другое расположение. Конкретные числовые факты Андреева — 729=3^6, 242=22²/2, 753=3×251 — суть точные суммы подмножеств треугольной матрицы: верифицируются алгоритмически, дают новые Q6-объекты.

**Уникальный результат.** Обнаружение, что матрица И-Цзин делится свастикой в пропорции 3:4:5 (832:1248:2080) — это конкретный числовой факт, верифицируемый программно. Внутри него: левая свастика (753) происходит из бинарной структуры 2^6, тройная симметрия — из D₃-орбит треугольника. Это связывает hexsym (D₄/D₃ орбиты) с hexnav (гексаграммы) через новую геометрию.

**Команды:**
```
hextrimat triangle        # 64 гексаграммы в треугольных 11 строках
hextrimat bird            # подсветить «Птицу Времени» (сумма=729=3^6)
hextrimat thoth           # «Цветок Тота» (242=22²/2)
hextrimat swastika        # левосторонняя свастика (753) и деление 3:4:5
hextrimat twins           # 13 пар «близнецов» с суммами
hextrimat d3              # D₃-орбиты: вращение треугольника на 120°/240°
hextrimat center          # ячейка 27 — «главная Инь», центр вращения
```

## Часть II — Карта синергий: что создают пары и тройки

> Синергия = когда два модуля вместе дают результат X,
> который **ни один из них не может получить по отдельности**.

---

### Синергия 1: `hexcrypt` × `hexring` → «Платиновые S-блоки»

**Что умеет каждый отдельно:**
- `hexcrypt`: строит S-блоки с измерением NL, DDT, LAT, SAC — но использует _случайные_ или _эвристические_ S-блоки.
- `hexring`: находит все изогнутые (bent) булевы функции Q6 с максимальной нелинейностью NL=24 — но не строит шифры.

**Что возникает вместе:**
Bent-функция из `hexring` подаётся как S-блок в `hexcrypt`. Результат: S-блок с **доказуемо максимальной нелинейностью** (NL=24 = теоретический максимум для 6 переменных), проверенный на все криптографические критерии. Это называется «платиновый S-блок» — такого нет в SageMath, OpenSSL или любой другой библиотеке с открытым кодом.

```bash
# Пример пайплайна:
python -m hexring list-bent --yang 3 | python -m hexcrypt sbox-eval
# → Найти все bent-функции с yang=3 и оценить их как S-блоки
```

---

### Синергия 2: `hexca` × `hexstat` → «Статистический профиль автоматов»

**Отдельно:**
- `hexca`: 256 правил CA на Q6, эволюция, аттракторы.
- `hexstat`: энтропия, χ², KS-тест, взаимная информация — но нет генератора данных.

**Вместе:**
Классификация всех 256 правил CA по **статистическому профилю**: entropy(rule), period(rule), mixing_time(rule). Аналог классификации Вольфрама (классы 1,2,3,4), но сделанный для Q6 с точными числами. Правила с entropy → max: кандидаты в генераторы псевдослучайных чисел. Правила с mixing_time → min: быстрое перемешивание, применимо в shuffle-алгоритмах.

---

### Синергия 3: `hexca` × `hexstat` × `hexsym` → «Канонический атлас CA»

**Добавляем `hexsym`:**
Aut(Q6) сворачивает 256 правил в ~40 классов симметрии (правила, дающие одинаковый результат после перестановки битов).

**Тройной результат:**
«Канонический атлас CA на Q6» — таблица: для каждого из ~40 канонических классов: его энтропия, длина периода, класс Вольфрама, размер класса под действием Aut(Q6). Этого не существует нигде — такое исследование требует одновременно CA-теории, теории групп и статистики.

---

### Синергия 4: `hexcrypt` × `hexring` × `hexgf` → «Доказуемо стойкий шифр Q6»

**Три компонента:**
- `hexring`: bent-функции → слой подстановки (S-слой, нелинейность=24).
- `hexgf`: MDS-матрицы над GF(2^6) → слой перестановки (P-слой, branch number = max).
- `hexcrypt`: Feistel/SPN-сеть, ключевое расписание, измерение стойкости.

**Тройной результат:**
Полный блочный шифр на Q6 с **доказанными** (не эвристическими) параметрами:
- Нелинейность S-слоя = 24 (максимум).
- Branch number P-слоя = 7 (MDS-оптимум для 6 бит).
- Число раундов для достижения полной лавинности = вычислено точно.

Это уровень публикации в криптографическом журнале.

---

### Синергия 5: `hexphys` × `hexca` → «Термодинамика клеточных автоматов»

**Отдельно:**
- `hexphys`: модель Изинга на Q6, энергия H = −Σσᵢσⱼ, алгоритм Метрополиса, фазовые переходы.
- `hexca`: детерминированные правила эволюции — но нет температуры и энергии.

**Вместе:**
Каждое правило CA интерпретируется как гамильтониан. Энергия гексаграммы h под правилом r = «насколько стабильна конфигурация h». Применяем Метрополис из `hexphys` к пространству правил — «какая температура нужна, чтобы CA перешёл из упорядоченного режима в хаотический?» Это фазовый переход в пространстве CA-правил. Аналог точки Кюри, но для дискретных автоматов.

---

### Синергия 6: `hexlearn` × `hexopt` → «Байесовская оптимизация на Q6»

**Отдельно:**
- `hexlearn`: KNN, спектральная кластеризация, байесовский классификатор — пассивный анализ данных.
- `hexopt`: SA, генетический, табу-поиск — слепой поиск без модели ландшафта.

**Вместе:**
`hexlearn` строит **суррогатную модель** фитнес-ландшафта Q6 (gaussian process или random forest на 64 точках). `hexopt` использует эту модель для направленного поиска — вместо случайных шагов идём туда, где модель предсказывает улучшение (Expected Improvement). Число оценок целевой функции сокращается с O(2^64) до O(64·log64). Это Байесовская оптимизация — стандартная методология ML, применённая к Q6 с его конкретной метрикой Хэмминга.

---

### Синергия 7: `hexdim` × `hexbio` → «Геномный фрактальный отпечаток»

**Отдельно:**
- `hexbio`: ДНК-кодоны (A,C,G,U → 2 бита → 6 бит) вложены в Q6, мутационный граф.
- `hexdim`: фрактальная размерность подкубов, самоподобие, box-counting.

**Вместе:**
ДНК-последовательность → путь в Q6 (каждый кодон = вершина, каждая мутация = ребро) → измерить **фрактальную размерность** этого пути в Q6. Разные организмы имеют разные dim(Q6-пути). Гипотеза: эволюционно близкие организмы → близкие фрактальные размерности. Новый биоинформатический маркер. Ни один из существующих инструментов биоинформатики не использует Q6-фрактальную размерность.

---

### Синергия 8: `hexnav` × `hexspec` × `hexca` → «Автомат И-Цзин»

**Три компонента:**
- `hexnav`: 64 состояния-гексаграммы с триграммной структурой.
- `hexspec`: верификация конечных автоматов (достижимость, мёртвые состояния, пути).
- `hexca`: правила CA = детерминированные функции переходов.

**Тройной результат:**
«Какие правила CA превращают И-Цзин в эргодический автомат, обходящий все 64 гексаграммы?» `hexca` генерирует правило-переход, `hexspec` верифицирует достижимость всех 64 состояний, `hexnav` интерпретирует результат в терминах гексаграмм. Первая формальная верификация «Книги Перемен» как динамической системы.

---

### Синергия 9: `hexpowerxy` × `hexellipse` × `hexcrossrat` → «Золотой триптих»

**Три компонента:**
- `hexpowerxy`: кривая X^Y=Y^X, золотой корень при t=φ.
- `hexellipse`: семейство a=k+1, b=7−k; при k=3 → a=b=4 → окружность.
- `hexcrossrat`: R6≅S₃ с w=φ; инварианты Σrᵢ=3, ∏rᵢ=1.

**Тройной результат:**
Число φ является **глобальной неподвижной точкой** одновременно для всех трёх математических объектов. Кривая X^Y=Y^X имеет особую точку при t=φ; эллипс при yang=3 имеет a·b=16=4² (квадрат 4, ближайший к φ²×e); cross-ratio при w=φ даёт r₅=φ². Эти три системы «видят» φ независимо — это говорит о том, что φ является не просто константой, а структурным инвариантом Q6. Единственный способ это обнаружить — запустить три модуля вместе и сравнить особые точки.

---

### Синергия 10: `karnaugh6` × `hexcode` × `hexring` → «Схемы с кодовой защитой»

**Три компонента:**
- `karnaugh6`: минимизация булевых функций (Куайн-МакКласки), prime implicants.
- `hexcode`: коды Хэмминга/Рида-Мюллера, матрица проверки чётности H.
- `hexring`: изогнутые (bent) функции, Walsh-Hadamard спектр.

**Тройной результат:**
Найти булевы функции, которые одновременно: (1) имеют минимальное число термов (karnaugh6), (2) являются корректирующими кодами (hexcode), (3) максимально нелинейны (hexring). «Схема-с-защитой» — логическая схема с минимальным числом вентилей, исправляющая ошибки и устойчивая к линейному криптоанализу. Применение: безопасная аппаратная криптография. Такой дизайн-поиск требует всех трёх модулей одновременно.

---

### Синергия 11: `hexuniqgrp` × `hexperms` → «Уникальные перестановочные группы»

**Два источника (оба — работы Германа):**
- `hexuniqgrp`: группы уникального порядка (только одна группа данного порядка n).
- `hexperms`: все 720=6! перестановок {0..5} с факториальным номером.

**Вместе:**
Вопрос: «Какие перестановки битов Q6 порождают группы уникального порядка?» Для каждой из 720 перестановок σ вычисляем ord(⟨σ⟩) — порядок порождённой ею циклической группы — и проверяем по `hexuniqgrp` уникальность этого порядка. Возникает «карта уникальных генераторов» в S₆. Объединение двух работ Германа в одну Q6-задачу — то, что сам Герман, возможно, не формулировал.

## Итог: что «часовой механизм» из 39 модулей умеет делать группами

```
Криптографический кластер:
  hexring ──→ hexcrypt ──→ hexgf
                ↑               ↑
           (bent S-box)   (MDS диффузия)
           → Полный Q6-шифр с доказанными параметрами

Динамический кластер:
  hexca ──→ hexstat ──→ hexsym
    ↑                      ↑
(правило)            (симметрия)
           → Канонический атлас CA

Интеллектуальный кластер:
  hexlearn ──→ hexopt
      ↑            ↑
  (модель)    (поиск)
           → Байесовская оптимизация

Биологический кластер:
  hexbio ──→ hexdim
     ↑           ↑
  (геном)   (фрактал)
           → Геномный отпечаток

Германовский кластер (два его модуля вместе):
  hexperms ──→ hexuniqgrp
     ↑               ↑
(перестановки)  (уникальные ord)
           → Уникальные генераторы в S₆
```

Каждый кластер производит результат, недостижимый одним модулем.
Именно такие комбинации делают 39 модулей **системой**, а не просто набором.
