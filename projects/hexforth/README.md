# hexforth — Стековый язык на графе Q6

Язык программирования в духе Forth, где каждое слово — гексаграмма,
каждая программа — путь через Q6, исполнение — навигация по рёбрам графа.

## Статус

Идея → исследование

## Ключевая связь с Forth

```
Forth оперирует стеком: [ A | B | C | D | E | F ]
                                ↓
6 слотов стека = 6 бит = гексаграмма

Слово Forth → меняет состояние стека → переход в Q6
```

## Что это даёт

- **Формальная верификация**: достижимость состояния = поиск пути в Q6
- **Визуальный дебаггер**: текущее состояние стека видно как гексаграмму
- **Минимальный словарь**: 64 «слова» = 64 гексаграммы
- **Корректность по конструкции**: только переходы по рёбрам Q6 допустимы

## Связь с другими стековыми системами

- WebAssembly (стековая VM)
- JVM bytecode
- PostScript
- RPL (калькуляторы HP)

Для всех них hexforth может служить формальной семантикой подмножества операций.

## Стек (языка реализации)

Python 3.10+ (интерпретатор прототипа). Потенциально: реализация на самом Forth.

## Пример (концептуальный)

```
# Программа: переход из гексаграммы 0 в гексаграмму 42
0 START
FLIP-0   # перевернуть черту 0: 000000 → 000001
FLIP-1   # перевернуть черту 1: 000001 → 000011
FLIP-3   # перевернуть черту 3: 000011 → 001011
FLIP-5   # перевернуть черту 5: 001011 → 101011 = 43
# ... путь определяется автоматически через shortest_path(0, 42)
```
