# Q6 Glyph Module Ecosystem — Полный каталог

> 39 модулей визуализации для 6-мерного гиперкуба Q6
> Версия 1.0 · Ветка `claude/review-pdf-document-qvUbL`

---

## 1. Что это такое

**Q6 Glyph Ecosystem** — коллекция из 39 Python-модулей, каждый из которых
отображает математическую или инженерную предметную область
на единую визуальную систему из 64 символов-«глифов».

Каждый глиф — вершина 6-мерного булева гиперкуба Q6 (число от 0 до 63).
Его 6 битов управляют 6 сегментами маленького квадратного значка:

```
 _       ← bit 0 (верхняя черта)
|X|      ← bit 2 (лево), bit 4+5 (диагональ), bit 3 (право)
 _       ← bit 1 (нижняя черта)
```

Вся коллекция укладывается в **8×8 решётку Gray-кода**: соседние ячейки
отличаются ровно одним битом — то есть соединены ребром Q6.

---

## 2. Уникальность

| Свойство | Описание |
|---|---|
| **Единая визуальная система** | 39 разных предметных областей — одно и то же 8×8 поле |
| **Математическая строгость** | Каждое отображение — доказуемый изоморфизм или гомоморфизм |
| **Нулевые зависимости** | Только стандартная библиотека Python + `libs/hexcore` |
| **Терминальный вывод** | ANSI 256-цветная сетка прямо в консоли, без GUI |
| **64 = 2⁶** | Число 64 встречается в генетике (кодоны), криптографии (S-box), физике (конфигурации), информатике (6-битные коды) — всё это отображено |
| **Gray-код структура** | Соседние глифы = соседние состояния ⟹ переходы видны визуально |

## 3. Кому применять

### 3.1 Исследователь / математик
Видит структуру объекта через пространственное расположение.
Например: орбиты Aut(Q6) выглядят как концентрические полосы,
группа R6 — как 6 цветовых зон.

### 3.2 Инженер-программист
Отлаживает алгоритмы на Q6: маршрутизацию, клеточные автоматы,
верификацию конечных автоматов. Каждый модуль — готовый CLI.

### 3.3 Специалист по криптографии / кодированию
Анализирует S-блоки, синдромные коды, GF(2⁶).
Видит дифференциальные и линейные характеристики наглядно.

### 3.4 Биоинформатик
64 кодона = 64 глифа. Синонимичные кодоны — кластеры.
Мутационные расстояния — расстояния Хэмминга в Q6.

### 3.5 Педагог / студент
Интерактивный инструмент для объяснения теории групп,
теории кодирования, топологии, теории информации.
Каждая команда — наглядная иллюстрация теоремы.

### 3.6 Разработчик игр / головоломок
`hexpath` — двухигровая задача на Q6 с ИИ-противником.
Встроенные головоломки (11 штук) + minimax AI.

```
Конкурент                 Разрыв с Q6-экосистемой
─────────────────────────────────────────────────────────
SageMath / Mathematica    Тяжёлые зависимости, нет единой
                          визуальной системы для 64-элем. объектов

NetworkX                  Только теория графов; нет физики,
                          генетики, топологии, криптографии

numpy + matplotlib        Требуют GUI; нет 8×8 терминального ASCII

Cube / Penrose tilers     Только геометрия, нет других доменов
```

**Ниша**: лёгкий (zero-dep) терминальный инструмент
для математического исследования на пространстве из ровно 64 элементов
с единым визуальным языком across 39 предметных областей.

#### `hexsym/sym_glyphs.py` — Группа автоморфизмов Aut(Q6)

**Описание.** Aut(Q6) — группа порядка 46 080 = 2⁶·6!, действующая

**Функции CLI:**

|---|---|

**Кому:** алгебраисты, теоретики кодирования, студенты теории групп.
**Уникальность:** |Aut(Q6)| = 46 080 показан через конкретные орбиты на 8×8 сетке.

**Описание.** R6 ≅ S₃ — 6 преобразований Мёбиуса:

## 7. Типичные вопросы и ответы (FAQ)

**Q: Что такое Q6 и почему именно 64 элемента?**
A: Q6 — 6-мерный булев гиперкуб: все 6-битные числа 0..63.
Число 64 = 2⁶ встречается в десятках областей одновременно:
генетика (64 кодона), криптография (6-битные S-блоки), теория кодирования
(коды длины 6), физика (2⁶ спиновых конфигураций). Это делает Q6
«универсальной площадкой» для математических экспериментов.

---

**Q: Нужно ли устанавливать дополнительные библиотеки?**
A: Нет. Все модули используют только стандартную библиотеку Python 3.10+
и внутренний `libs/hexcore`. Никаких numpy, matplotlib, scipy.

---

**Q: Как запустить любой модуль?**
A:
```bash
python -m projects.<проект>.<модуль> <команда> [--опции]
# Пример:
python -m projects.hexbio.codon_glyphs codons
python -m projects.hexcrypt.sbox_glyphs ddt --sbox identity
python -m projects.hexphys.spin_glyphs energy
```

---

**Q: Что значит цвет в 8×8 сетке?**
A: По умолчанию цвет = ян-уровень `yang_count(h)` = число единиц в двоичном
представлении h. Семь уровней (0..6) окрашены от тёмно-серого (0) до
ярко-жёлтого (6). Каждый модуль может переопределить цвет для
своей предметной области (например, hexcrypt красит по DDT-максимуму).

---

**Q: Можно ли отключить цвет?**
A: Да, флаг `--no-color` есть у всех 39 модулей:
```bash
python -m projects.hexsym.sym_glyphs --no-color yang
```

---

**Q: Как связаны разные модули между собой?**
A: Все они используют одно пространство — вершины Q6 (числа 0..63).
Это значит, что результаты одного модуля можно «наложить» на другой.
Например: найти bent-функции в `hexring`, затем проверить их как S-блоки
в `hexcrypt`, затем исследовать их орбиты в `hexsym`.

---

**Q: Что такое Gray-код в контексте 8×8 сетки?**
A: Строки сетки = биты 5,4,3 в Gray-коде [0,1,3,2,6,7,5,4].
Столбцы = биты 2,1,0 в том же Gray-коде. Это гарантирует, что
соседние ячейки (по горизонтали и вертикали) отличаются ровно одним битом —
то есть соединены ребром Q6. Топология сетки = топология Q6.

---

**Q: Для каких учебных курсов подходит экосистема?**
A: Дискретная математика, теория групп, теория кодирования,
криптография, теория информации, топология, комбинаторика,
алгоритмы и структуры данных, биоинформатика, физика конденсированного
состояния (модель Изинга), машинное обучение (метрические классификаторы).

---

**Q: Можно ли добавить свой модуль?**
A: Да. Шаблон: создайте `projects/<hexXXX>/<domain>_glyphs.py`.
Импортируйте из `libs.hexcore.hexcore` функции `yang_count`, `neighbors`,
`hamming`. Импортируйте `_YANG_ANSI`, `_RESET`, `_BOLD` из
`projects.hexvis.hexvis`. Реализуйте 4 функции `render_*` и `main()`.

---

**Q: Какова производительность при больших вычислениях?**
A: 64 вершины — очень малое пространство. Большинство операций (BFS, орбиты,
WHT, DDT) выполняются за миллисекунды. Исключения: Monte-Carlo симуляции
(`hexbuffon`, n=10⁶) и полный перебор (`hexcrypt` для случайных S-блоков).

---

**Q: Есть ли тесты?**
A: Каждый модуль содержит smoke-тесты в виде флага `--no-color`
+ конкретная команда. Инварианты (Σrᵢ=3, a·b=c·p, X^Y=Y^X) проверяются
прямо в выводе модуля (`✓`/`✗`).

---

**Q: Можно ли использовать модули как библиотеку (не CLI)?**
A: Да. Каждая `render_*` функция принимает `color: bool` и возвращает строку.
Можно вызывать напрямую:
```python
from projects.hexphys.spin_glyphs import render_energy
print(render_energy(color=False))
```

---

**Q: Какой модуль лучше всего демонстрировать новичкам?**
A: `hexvis/vis_glyphs.py hasse` — диаграмма Хассе B₆ сразу показывает
всю структуру Q6 одним экраном. `hexbio/codon_glyphs codons` — для
биологов мгновенно понятна связь 64 кодонов с 64 глифами.

---

**Q: Где математическое обоснование изоморфизмов?**
A: В заголовочных docstring каждого `*_glyphs.py` файла.
Например, `hexcrossrat/crossrat_glyphs.py` строки 1-40 объясняют R6≅S₃
и инварианты Σrᵢ=3, Πrᵢ=1. Источники: работы Ф. Германна.

---

**Q: Есть ли модуль для случайных блужданий или марковских цепей?**
A: `hexopt/landscape_glyphs walk` и `hexca/ca_glyphs evolve` ближе всего.
Марковская цепь на Q6 с равновероятными переходами по рёбрам —
стационарное распределение равномерное (Q6 регулярен).

---

**Q: Что такое «антипод» и почему он важен?**
A: Антипод(h) = h XOR 63 = инверсия всех 6 битов. Расстояние Хэмминга
между h и антиподом всегда равно 6 (максимум для Q6). Антиподальные пары
{h, h⊕63} разбивают Q6 на 32 пары. Встречается в `hexsym`, `hexnav`,
`hexmobius`, `hexscrew`.

---

**Q: Почему yang_count используется как главный параметр?**
A: `yang_count(h) = popcount(h)` = число единиц в двоичном h.
Это единственный инвариант группы Aut(Q6): автоморфизмы не меняют
yang_count. Поэтому любая Aut(Q6)-инвариантная функция зависит
только от yang_count — 7 возможных значений (0..6).
