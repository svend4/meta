# Сравнительный анализ: PDF-коллекция vs Экосистема Q6

**Дата:** 19 февраля 2026 г.
**Охват:** 33 PDF-документа (Франц Герман) + 24 проекта экосистемы Q6

---

## 1. Что такое каждая коллекция

### 1.1 PDF-коллекция (33 работы Франца Германа)

Единоличный авторский математический проект. Автор на протяжении многих лет разрабатывает оригинальные конструкции в чистой математике: теория групп, проективная геометрия, топология, теория чисел. Работы направлялись в РАН, рецензирования не проходили. **Нет ни одной реализации в коде.**

### 1.2 Экосистема Q6 (24 проекта)

Монорепо на Python, построенное вокруг одной математической структуры — **графа Q6** (6-мерный гиперкуб, 64 вершины). Все 24 проекта используют `libs/hexcore` как ядро и реализуют разные области: от клеточных автоматов и языков программирования до биоинформатики и криптографии. **Всё реализовано в коде.**

## 2. Математические основы

### 2.1 Граф Q6 (основа экосистемы)

```
Вершины: все 6-битные числа {0,1}⁶ → 64 гексаграммы (И-Цзин)
Рёбра:   два узла соединены ↔ отличаются ровно в 1 бите
Степень каждой вершины: 6
Диаметр графа: 6
Группа симметрий: Aut(Q6) = B₆ = S₆ ⋉ (Z₂)⁶, порядок = 46 080
```

Q6 — это **граф Кэли группы (Z₂)⁶**. Он возникает в теории кодирования (коды Хэмминга), квантовых вычислениях, синтезе цифровых схем (карты Карно).

### 2.2 Математика из PDF-коллекции

PDF-коллекция не опирается на единую структуру. Каждая работа исследует свою конструкцию:
- Группа-винт Bₙ ⊂ Sₙ (e2c)
- Проективные конфигурации (1da, 566, 46c)
- Теория чисел: промежуточный ряд, генетическая теория (25d, 6a4)
- Топология замкнутых поверхностей (65b, 80e, 481)
- Апериодические мозаики (0f6, 194)
- Замкнутые клеточные поля (6ec)

## 5. Клеточный автомат: сравнение hexca vs PDF-работы

### 5.1 hexca — что реализовано

```
Тип:        1D лента (периодические граничные условия)
            2D тор (W×H с торической топологией)
Состояния:  64 (гексаграммы Q6) — vs 2 у Вольфрама, vs 2 у Конвея
Правила (встроенные):
  majority_vote  — переворот черты если большинство соседей изменили её
  xor_rule       — XOR текущего со всеми соседями
  conway_like    — порог активации по числу ян-линий
  identity       — стабильный паттерн
  random_walk    — случайный сосед
Визуализация: ASCII по числу ян-линий (0 ян = пусто, 6 ян = █)
```

### 5.2 PDF 6ec — Упаковки замкнутых клеточных полей

```
Тип:        Замкнутое клеточное поле (тороидальная решётка)
Состояния:  Числовые значения (не бинарные)
Правила:    Алгоритмические правила заполнения без перекрытий
Инварианты: Симметрии замкнутого поля
```

### 5.3 Чего не хватает hexca (и как PDF могут помочь)

| Дефицит hexca | Источник из PDF | Описание |
|---------------|-----------------|----------|
| Бедный набор правил (5 штук) | **e2c** — Группа-винт Bₙ | Действие Bₙ на гексаграммах → систематическое порождение правил переходов |
| Нет алгебраических правил | **6ec** — Клеточные поля | Правила упаковки → правила перехода с алгебраическим инвариантом |
| Нет квазикристаллических IC | **0f6 + 194** — Мозаики | Апериодические паттерны → начальные конфигурации 2D hexca |
| Нет топологических вариантов | **80e** — Замкнутые маршруты | Другие топологии (цилиндр, лист Мёбиуса) для решётки CA |
| Нет обратимых правил | **6cf** — Цикл. изоморфизм | Циклически-изоморфные правила = обратимые CA |

## 6. Уникальность каждого блока

### 6.1 Уникальность PDF-коллекции

| Уровень | Работы | Обоснование |
|---------|--------|-------------|
| **Высокая** | e2c, 1da, 6a4, 25d | Конкретные новые конструкции с доказательствами; прямых аналогов в литературе не выявлено |
| **Средняя** | 0f6, 6cf, a67, 6ec, 65b, bb1 | Оригинальные идеи, частично пересекающиеся с известным |
| **Ниже средней** | e5b, 706, 927, 16f, e05 | Правильные, но переоткрытые или спекулятивные утверждения |

**Главная слабость**: нет реализаций, нет рецензирования, нет сопоставления с существующей литературой.

## 10. Синергия: как объединить

### 10.1 Три конкретных улучшения hexca на базе PDF

**Улучшение 1: Правила Bₙ для hexca** (PDF e2c)
```
Текущее состояние: 5 правил вручную
Новый подход: использовать действие группы-винта Bₙ на {0..63}
как систематический генератор правил переходов.
Результат: тысячи новых, алгебраически обоснованных правил
```

**Улучшение 2: Квазикристаллические начальные конфигурации** (PDF 0f6 + 194)
```
Текущее состояние: случайная или константная инициализация
Новый подход: разместить апериодические тайлы автора в сетке,
закодировав тип плитки как состояние гексаграммы (0..63)
Результат: наглядная эволюция квазикристаллических паттернов
```

**Улучшение 3: Тороидальная упаковка** (PDF 6ec)
```
Текущее состояние: правила независимы от глобального состояния
Новый подход: добавить правило "cellular_packing" из PDF 6ec
— поддерживать инвариант суммы состояний в замкнутом поле
Результат: консервативный CA с сохраняющимся квантом
```

### 11.2 Главные выводы

1. **PDF-коллекция и Q6-экосистема дополняют, а не дублируют друг друга.** PDF даёт математические идеи без реализации; Q6 — реализацию без ряда оригинальных математических концепций PDF.

2. **hexca (клеточные автоматы) — точка максимального пересечения.** PDF 6ec, 0f6, 194, e2c, 6cf — все могут напрямую обогатить hexca новыми правилами и начальными конфигурациями.

3. **Самые уникальные проекты Q6** — karnaugh6 (нет аналога для 6 переменных), hexbio (DNA на Q6), hexforth (язык программирования на графе), hexspec (верификатор через Q6).

4. **Самые уникальные PDF** — e2c (группа-винт), 1da (линии Паскаля), 6a4 (генетическая теория чисел), 25d (промежуточный ряд).

5. **Коллизия имён B₆**: в hexsym B₆ = S₆ ⋉ (Z₂)⁶ (hyperoctahedral group), в PDF e2c Bₙ = другая конструкция. Это важно прояснить во избежание путаницы.

6. **Нерализованный потенциал PDF**: если реализовать идеи работ e2c и 6ec в Python, их можно напрямую включить в hexsym и hexca соответственно.
