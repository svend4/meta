# meta

## Какой это агент и модель

- Агент: GitHub Copilot Coding Agent (ассистент для задач разработки).
- Модель: LLM из семейства OpenAI; точная версия может зависеть от конфигурации среды выполнения.
- Что умеет: анализировать требования, работать с кодом и документацией, предлагать и вносить минимальные изменения, запускать доступные проверки и объяснять результат.
- Какие функции выполняет: поиск причин ошибок, исправление дефектов, небольшие доработки, обновление тестов/документации и сопровождение изменений до проверяемого результата.

## Возможности агента

Агент помогает решать технические и организационные задачи в разработке ПО:

- анализирует задачу и уточняет требования;
- изучает структуру проекта и существующий код;
- предлагает и вносит минимальные, точечные изменения;
- пишет и корректирует код с учетом стиля проекта;
- помогает с отладкой, поиском причин ошибок и регрессий;
- добавляет или обновляет тесты в рамках существующей инфраструктуры;
- проверяет изменения через доступные команды сборки/линтинга/тестов;
- формирует понятные отчеты о ходе работ и результате.

## В каких задачах помогает

- исправление багов и дефектов;
- реализация небольших функциональных доработок;
- улучшение документации и описаний;
- рефакторинг без изменения внешнего поведения;
- анализ падений CI и локализация проблем.

## Какие приложения сейчас востребованы и как с ними помочь

На практике высокий спрос обычно есть на прикладные решения, которые экономят время и деньги: веб-сервисы для бизнеса, автоматизация внутренних процессов, аналитические панели, мобильные клиентские приложения, а также утилиты и интеграции между системами.

Если есть подтвержденный спрос (запросы пользователей, задачи команды, боли клиентов), агент помогает довести идею до работающего решения: уточнить требования, разложить проект на этапы, подготовить архитектуру и структуру проекта, реализовать ключевые функции, добавить/обновить тесты в рамках существующей инфраструктуры, провести проверку изменений и оформить результат понятной документацией.

Итоговый формат помощи: от прототипа (MVP) до точечных улучшений уже существующей программы — с фокусом на минимальные, проверяемые и безопасные изменения.

## Как работает

1. Сначала изучает контекст задачи и текущее состояние репозитория.
2. Определяет минимальный безопасный объем изменений.
3. Вносит правки и проверяет, что они решают исходную проблему.
4. Запускает релевантные проверки (если они есть в проекте).
5. Возвращает результат с объяснением, что изменено и почему.

## Пример программы: магазин цветов

В репозиторий добавлено минимальное CLI-приложение `flower_shop.py`.

Запуск:

```bash
python3 flower_shop.py
```

Возможности программы:
- просмотр каталога цветов;
- добавление в корзину;
- удаление из корзины;
- просмотр итоговой суммы;
- оформление заказа.
